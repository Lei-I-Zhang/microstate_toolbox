+microstate toolbox basics

This documentation will go over some basics of the +microstate toolbox. The +microstate toolbox was written for microstate analysis of EEG or MEG data in sensor- or source-space. Algorithms used by +microstate are described in [1,2]. Please cite these papers when using the +microstate toolbox.

[1] Tait and Zhang (2021) +microstate: A Matlab toolbox for brain microstate analysis in sensor and cortical EEG/MEG, in preparation

[2] Tait and Zhang (2021) Cortical imaging of brain electrophysiological microstates from source space magnetoencephalography, in preparation

Installation

+microstate runs in Matlab, and is compatible with Matlab R2017b or higher. +microstate can be downladed from the toolbox Github page. To use the toolbox, the folder +microstate must be in a directory on the Matlab path. However, +microstate itself (as a package folder) should not be added to the path.

The "individual" object

The microstate.individual object stores data from an individual scan/trial etc. An empty individual object can be called using the command below:

ms = microstate.individual ;

The microstate.individual object has various properties. For example, ms.data contains the MEG/EEG/source/amplitude envelope data. It also has many different methods. For example, ms = ms.preprocess_filter(1,30) will bandpass filter the data in the 1-30 Hz band. A list of properties and methods associated with the microstate.individual class are included in the documentation, which is accessed via the following command:

doc microstate.individual

Some simple functions are:

ms = ms.add_data(data,modality,time) ; % add data to the object
ms = ms.preprocess ; % run a default preprocessing pipeline
ms = ms.cluster_estimatemaps(k) ; % perform microstate analysis for k states
ms = ms.cluster_koptimum ; % perform microstate analysis for 2-20 states and select the optimum
ms = ms.networks_wpli ; % calculate microstate functional connectivity
ms = ms.stats_all ; % calculate microstate statistics
ms = ms.simulate ; % simulate data using a default pipeline
ms.plot('string') ; % visualization

Some of the main properties of interest are:

ms.maps % the microstate maps
ms.label % the microstate sequence
ms.stats % a structure of microstate statistics

The "cohort" object

The microstate.cohort object stores data from multiple scans/trials. An empty cohort object can be called using the command below:

coh = cohort.individual ;

Like the microstate.individual object, the the microstate.cohort object also has various properties and methods. A list of properties and methods associated with the microstate.cohort class are included in the documentation, which is accessed via the following command:

doc microstate.cohort

Some simple functions are:

coh = coh.add_individuals(individual,condition,storage) ; % add individual to the object
coh = coh.cluster_global(k) ; % perform group-level microstate analysis for k states
coh = coh.cluster_globalkoptimum ; % perform group-level microstate analysis for 2-20 states and select the optimum
coh = coh.ind_stats_all ; % calculate statistics of microstates for each individual
coh = coh.cohort_stats ; % concatenate individual statistics to group level
coh.plot('string') ; % visualization

Some of the main properties of interest are:

coh.globalmaps % the group-level microstate maps
coh.stats % a structure of group-level microstate statistics
